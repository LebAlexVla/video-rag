# ADR-004 — Связь C# и Python строится через запуск процесса и JSON-файл

- **ID:** ADR-004
- **Статус:** Принято

## Контекст

Внутренний HTTP между C# и Python для MVP создаёт лишнюю сложность: второй сервис, сетевые ошибки, контракты API, отдельный жизненный цикл.

## Решение

C# запускает Python как внешний процесс.

Обмен данными идёт через:
- аргументы командной строки;
- код завершения процесса;
- `transcript.json`.

Внутренний REST, gRPC, вебхуки и очереди между C# и Python не используются.

## Последствия

- Граница между частями системы формализована, но остаётся простой.
- Взаимодействие не зависит от сети.
- Упрощается локальный запуск.

## Что это значит для реализации

Разрешено:
- `Process.Start(...)` или аналог;
- фиксированный формат JSON-выхода;
- проверка exit code и обработка ошибок.

Запрещено:
- поднимать FastAPI для внутреннего обмена;
- вызывать Python через HTTP localhost;
- добавлять промежуточный транспорт.

Откладывается:
- API-взаимодействие между частями системы при росте сложности.

## Какие варианты были отвергнуты

- REST между C# и Python.
- Очередь задач между C# и Python.
- Общая оперативная память или нестабильное межпроцессное состояние.
