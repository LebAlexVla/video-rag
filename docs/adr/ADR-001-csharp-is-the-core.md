# ADR-001 — C# является центром системы

- **ID:** ADR-001
- **Статус:** Принято

## Контекст

Для MVP нужен один основной контур управления. Система должна быть понятна команде, быстро собираться и не распадаться на несколько равноправных частей.

## Решение

Основное приложение реализуется на C# / .NET.

C# отвечает за:
- вход в систему;
- команды загрузки лекций;
- чтение транскрипта;
- нарезку на чанки;
- построение векторов;
- запись в Qdrant;
- поиск контекста;
- вызов LLM;
- возврат ответа.

Python не управляет системой и не содержит логики RAG.

## Последствия

- Есть одна главная точка управления.
- Основная бизнес-логика не дублируется.
- Все ключевые сценарии проходят через C#.

## Что это значит для реализации

Разрешено:
- держать orchestration в C#;
- реализовывать ingestion, query и rebuild как сервисы C#.

Запрещено:
- переносить логику поиска или генерации ответа в Python;
- делать Python вторым центром системы.

Откладывается:
- возможное перераспределение ролей между языками после MVP.

## Какие варианты были отвергнуты

- Python как центр системы.
- Два равноправных backend-контура на C# и Python.
