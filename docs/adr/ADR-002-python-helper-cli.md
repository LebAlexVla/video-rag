# ADR-002 — Python используется как вспомогательный CLI для транскрибации

- **ID:** ADR-002
- **Статус:** Принято

## Контекст

Для транскрибации practical choice — использовать Python-инструменты. При этом отдельный Python backend для MVP не нужен.

## Решение

Python используется только как внешний CLI-скрипт.

Он:
- принимает путь к локальному видео;
- запускает транскрибацию;
- формирует `transcript.json`.

На этом его зона ответственности заканчивается.

## Последствия

- Python остаётся узким техническим инструментом.
- Межъязыковая граница простая и контролируемая.
- Основная архитектура не зависит от Python-сервиса.

## Что это значит для реализации

Разрешено:
- запускать Python как внешний процесс;
- получать от него JSON-артефакт.

Запрещено:
- добавлять в Python поиск, индексацию, Qdrant, пользовательские сценарии;
- превращать Python в HTTP-сервис.

Откладывается:
- расширение helper до downloader и других задач.

## Какие варианты были отвергнуты

- Полная реализация транскрибации на C#.
- Python как отдельный backend/API.
