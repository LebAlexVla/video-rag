# ADR-007 — В MVP используются только приблизительные ссылки на место в лекции

- **ID:** ADR-007
- **Статус:** Принято

## Контекст

Для первой версии важнее надёжный ответ с указанием источника, чем точная навигация до слова. Поддержка word-level timestamps увеличивает сложность транскрипта, chunking и хранения метаданных.

## Решение

В MVP система хранит и показывает:
- название лекции;
- индекс или номер чанка;
- приблизительное место в лекции;
- при наличии — примерный диапазон времени.

Таймкоды до слова и точная сегментная навигация не поддерживаются.

## Последствия

- Упрощается ingestion pipeline.
- Уменьшается объём метаданных.
- Источники остаются полезными, но не идеально точными.

## Что это значит для реализации

Разрешено:
- хранить `approxMinute`, `chunkIndex`, приблизительный диапазон;
- показывать приблизительную привязку к месту в лекции.

Запрещено:
- проектировать MVP вокруг word-level timestamps;
- требовать от интерфейса точного перехода к слову.

Откладывается:
- точная навигация по сегментам и словам.

## Какие варианты были отвергнуты

- Полная поддержка word-level timestamps в первой версии.
- Полный отказ от любой привязки к месту в лекции.
