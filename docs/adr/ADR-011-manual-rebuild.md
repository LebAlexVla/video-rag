# ADR-011 — Пересборка индекса выполняется вручную

- **ID:** ADR-011
- **Статус:** Принято

## Контекст

На MVP объём данных маленький, а администратор присутствует в контуре эксплуатации. Автоматическая переиндексация и миграции усложняют систему без заметной пользы.

## Решение

Пересборка индекса выполняется вручную по команде администратора.

Это основной способ:
- исправить индекс;
- переиндексировать данные после изменения настроек;
- очистить базу после ошибочной загрузки.

## Последствия

- Эксплуатация остаётся простой и контролируемой.
- Не нужна отдельная фоновая логика миграций индекса.

## Что это значит для реализации

Разрешено:
- команда `rebuild`;
- пересоздание коллекции и повторная индексация вручную.

Запрещено:
- автоматический reindex по изменению конфигурации;
- скрытая переиндексация «сама по себе».

Откладывается:
- автоматическое обновление индекса;
- миграции между версиями embeddings без ручного шага.

## Какие варианты были отвергнуты

- Автоматическая пересборка индекса.
- Полный отказ от rebuild как операции.
