# ADR-008 — Повторная загрузка и дедупликация видео не входят в MVP

- **ID:** ADR-008
- **Статус:** Принято

## Контекст

Обработка дублей требует отдельной политики идентификации, обновления и переиндексации. Для MVP это не основной сценарий.

## Решение

Специальная обработка повторной загрузки не реализуется.

Если лекция загружена повторно, это считается вне основного пользовательского пути. Для исправления состояния используется ручная пересборка базы.

## Последствия

- Упрощается ingestion.
- Администратор отвечает за аккуратность загрузки на этапе MVP.

## Что это значит для реализации

Разрешено:
- не проверять уникальность видео;
- считать повторный ingest административной ошибкой или вне сценария.

Запрещено:
- строить отдельную подсистему dedup/versioning в MVP.

Откладывается:
- хэширование файлов;
- реестр загруженных видео;
- политики `skip/update/replace`.

## Какие варианты были отвергнуты

- Автоматическая дедупликация по хэшу.
- Автоматический update существующей лекции.
- Версионирование видео и индекса в первой версии.
