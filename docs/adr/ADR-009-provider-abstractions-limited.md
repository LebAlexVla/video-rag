# ADR-009 — Абстракции провайдеров сохраняются, но набор реализаций MVP ограничен

- **ID:** ADR-009
- **Статус:** Принято

## Контекст

Система должна допускать замену:
- провайдера транскрибации;
- провайдера векторов;
- языковой модели.

Но реализация всех комбинаций в MVP слишком дорога по времени и тестированию.

## Решение

В коде сохраняются интерфейсы провайдеров.

При этом в MVP поддерживается только минимальный рабочий набор реализаций. Полная матрица комбинаций не реализуется и не считается частью первой версии.

## Последствия

- Архитектура не запирает систему на один стек.
- Реальный объём работы остаётся ограниченным.

## Что это значит для реализации

Разрешено:
- иметь интерфейсы и 1–2 реализации на роль;
- выбирать один основной рабочий путь и один запасной.

Запрещено:
- обещать поддержку всех комбинаций «локально/внешний API»;
- раздувать MVP ради полной универсальности.

Откладывается:
- расширение матрицы провайдеров;
- тестирование всех комбинаций.

## Какие варианты были отвергнуты

- Полностью жёсткая реализация без интерфейсов.
- Полная матрица реализаций уже в MVP.
