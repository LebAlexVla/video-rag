# ADR-006 — Входом ingestion в MVP является локальный видеофайл

- **ID:** ADR-006
- **Статус:** Принято

## Контекст

Загрузка по ссылкам с видеоплатформ нестабильна и требует отдельного downloader-контура. Для MVP это повышает риск срыва сроков.

## Решение

В MVP ingestion начинается с локального файла видео.

Система принимает путь к файлу и не занимается скачиванием видео из внешних источников.

## Последствия

- Самый рискованный внешний контур исключён из первой версии.
- Сценарий ingestion становится контролируемым и воспроизводимым.

## Что это значит для реализации

Разрешено:
- команда `ingest <path-to-file>`;
- работа только с локальным файлом.

Запрещено:
- принимать ссылку и пытаться скачивать видео в MVP;
- встраивать нестабильный downloader в основной контур.

Откладывается:
- downloader как отдельная точка расширения.

## Какие варианты были отвергнуты

- Ingestion по Rutube/VK ссылке в первой версии.
- Поддержка и локального файла, и downloader одновременно в MVP.
