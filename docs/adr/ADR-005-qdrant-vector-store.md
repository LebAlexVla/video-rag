# ADR-005 — Qdrant используется как отдельное векторное хранилище

- **ID:** ADR-005
- **Статус:** Принято

## Контекст

Нужен простой и понятный механизм хранения embeddings и similarity search. Объём данных небольшой, запуск локальный.

## Решение

Для векторного поиска используется Qdrant.

Qdrant хранит:
- векторы чанков;
- текст чанка;
- метаданные лекции.

Qdrant не используется как общая рабочая база проекта.

## Последствия

- Векторный поиск изолирован в специальном хранилище.
- Операционные данные не смешиваются с retrieval-данными.

## Что это значит для реализации

Разрешено:
- хранить в Qdrant только retrieval-представление данных;
- искать top-k по вектору вопроса;
- пересоздавать коллекцию вручную при rebuild.

Запрещено:
- хранить в Qdrant статусы задач, историю чатов, конфигурацию;
- использовать Qdrant как замену обычной базе для всего подряд.

Откладывается:
- возможная замена или миграция хранилища после MVP.

## Какие варианты были отвергнуты

- Использование файлов без отдельного vector store.
- Введение полноценной SQL-схемы только ради embeddings.
- Смешивание operational data и retrieval data в одном слое без явной границы.
