# ADR-003 — MVP реализуется как локальный монолит, без микросервисов

- **ID:** ADR-003
- **Статус:** Принято

## Контекст

Проект делается за 2 недели, командой студентов, на небольшом объёме данных и под локальный запуск. В таких условиях микросервисы увеличивают стоимость интеграции без заметной пользы.

## Решение

Архитектура MVP — одно основное C# приложение, локальный Qdrant и внешний Python helper.

Не используются:
- отдельные внутренние сервисы;
- брокеры сообщений;
- очереди;
- фоновые распределённые воркеры.

## Последствия

- Систему проще запускать и отлаживать.
- Меньше точек отказа.
- Ниже интеграционный риск.

## Что это значит для реализации

Разрешено:
- держать всю прикладную логику в одном процессе C#;
- запускать локальные зависимости рядом.

Запрещено:
- дробить систему на `ingestion service`, `query service`, `admin service`;
- вводить очередь задач ради одного локального процесса.

Откладывается:
- сервисное разбиение после появления реальной эксплуатационной необходимости.

## Какие варианты были отвергнуты

- Микросервисная схема с отдельными backend-сервисами.
- Событийная архитектура с брокером сообщений.
