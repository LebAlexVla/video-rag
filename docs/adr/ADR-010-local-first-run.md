# ADR-010 — Первая версия запускается локально, а не в облаке

- **ID:** ADR-010
- **Статус:** Принято

## Контекст

MVP нужен для быстрой реализации и демонстрации. Облачный запуск требует отдельного контура эксплуатации: деплой, сетевые настройки, безопасность, доступность.

## Решение

Первая версия запускается локально на ноутбуке.

Публичное облачное развёртывание не входит в MVP.

## Последствия

- Снижается стоимость первой версии.
- Упрощается разработка и отладка.
- Операционные задачи не мешают довести основной сценарий до конца.

## Что это значит для реализации

Разрешено:
- ориентироваться на локальный запуск всех частей;
- использовать локальные пути, локальный Qdrant и локальные команды.

Запрещено:
- закладывать обязательный публичный сервер в MVP;
- делать архитектурные решения только ради облака.

Откладывается:
- облачный деплой;
- публичный доступ;
- production-эксплуатация.

## Какие варианты были отвергнуты

- Обязательный облачный запуск первой версии.
- Гибридная схема с обязательной облачной инфраструктурой уже в MVP.
