# Overview

## Что это за проект

Video Lecture RAG Assistant — локальная система для вопросов по видеолекциям.

Пользователь задаёт вопрос по загруженным лекциям. Система ищет релевантные фрагменты в транскрибированном материале и формирует ответ только на основе найденного контекста.

Главная цель MVP — дать простой и воспроизводимый путь:

```text
локальное видео -> транскрипт -> индекс -> вопрос -> grounded-ответ
```

## Что входит в MVP

В MVP входит:

- локальный запуск;
- ingest локального видеофайла;
- транскрибация видео;
- сохранение `transcript.json`;
- чтение транскрипта в C#;
- chunking в C#;
- построение embeddings;
- индексация chunks в Qdrant;
- вопрос по загруженным лекциям;
- ответ на основе найденного контекста;
- источники ответа:
  - название лекции;
  - номер чанка;
  - примерное место в видео;
- ручной rebuild индекса.

## Что не входит в MVP

В MVP не входит:

- загрузка видео по ссылкам;
- downloader для внешних видеоплатформ;
- дедупликация видео;
- автоматический reindex;
- очереди и брокеры сообщений;
- микросервисы;
- внутренний REST/gRPC между C# и Python;
- auth и роли пользователей;
- chat history;
- word-level timestamps;
- OCR слайдов и доски;
- production deployment;
- отдельный frontend-проект;
- полноценный продуктовый web UI.

## Основной пользовательский сценарий

1. Администратор запускает ingest локального видеофайла.
2. Система транскрибирует видео и подготавливает лекцию к поиску.
3. Пользователь задаёт вопрос по содержанию лекций.
4. Система находит релевантные фрагменты.
5. Система возвращает ответ или честный fallback, если контекста недостаточно.
6. Ответ содержит источники.

## Обязательные интерфейсы MVP

Обязательные интерфейсы:

- CLI `ingest`;
- Minimal API `/ask`.

Дополнительный интерфейс:

- локальный Razor UI для ручного тестирования и демонстрации.

Razor UI не является основным контрактом MVP. Он не должен заменять Minimal API и не должен влиять на архитектурные границы системы.

## Ключевые ограничения

Команда должна сохранять следующие правила:

- C# / .NET остаётся центром системы.
- Python используется только как CLI-helper для транскрибации.
- Python не выполняет chunking, embeddings, retrieval и answer generation.
- Qdrant используется только как vector store.
- Rebuild выполняется вручную.
- При смене embedding model/provider/vector space требуется пересборка индекса.
- Сначала доводится до рабочего состояния один основной локальный стек провайдеров.

## Definition of Done для MVP

MVP считается готовым, если:

1. Можно загрузить локальное видео через CLI `ingest`.
2. После ingest лекция доступна для поиска.
3. Minimal API `/ask` принимает вопрос.
4. Система возвращает полезный ответ по найденному контексту или честный fallback.
5. В ответе есть источники: лекция и примерное место в видео.
6. Rebuild может пересобрать индекс из сохранённых transcript-файлов.
7. Проект можно локально запустить по `docs/quick-start.md`.
