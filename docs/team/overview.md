# Overview

## Назначение проекта

Video Lecture RAG Assistant — локальная система для вопросов по видеолекциям.

Система принимает локальный видеофайл, получает транскрипт, индексирует фрагменты лекции и позволяет задавать вопросы по загруженным материалам. Ответ формируется на основе найденного контекста и возвращается с источниками.

Основной сценарий:

```text
локальное видео -> транскрипт -> индекс -> вопрос -> ответ с источниками
```

## Пользовательский сценарий

1. Администратор запускает ingest локального видеофайла.
2. Python helper создаёт transcript JSON.
3. Основное C# приложение читает transcript, нарезает его на chunks и индексирует их в Qdrant.
4. Пользователь задаёт вопрос через API, UI или внешний клиент.
5. Система ищет релевантные chunks.
6. Answer provider формирует ответ на основе найденного контекста.
7. Пользователь получает ответ и ссылки на источники.

Если релевантного контекста недостаточно, система возвращает fallback-сообщение вместо уверенного ответа.

## Основные способы использования

### CLI

CLI используется для административных операций.

```text
ingest   загрузить и проиндексировать лекцию
rebuild  пересобрать индекс из сохранённых transcript-файлов
```

### HTTP API

HTTP API — основной контракт для вопросов и внешних клиентов.

```text
POST /ask
GET  /health
```

### UI и клиенты

Интерфейсный слой может быть представлен несколькими вариантами:

- встроенный Razor UI для локального тестирования;
- Telegram bot;
- отдельный Web UI;
- другие thin clients.

Клиенты обращаются к основному HTTP API и не реализуют RAG pipeline самостоятельно.

## Основные компоненты

### Main C# application

Центральная часть проекта. Управляет сценариями ingest, ask и rebuild.

Отвечает за:

- CLI и HTTP API;
- orchestration сценариев;
- запуск Python helper;
- чтение transcript;
- chunking;
- embeddings;
- запись и поиск в Qdrant;
- генерацию ответа через provider abstraction.

### Python helper

Вспомогательный CLI-инструмент для транскрибации.

Его задача — получить локальный видеофайл и создать transcript JSON, который затем обрабатывает C# приложение.

### Qdrant

Векторное хранилище для поиска релевантных фрагментов лекций.

Хранит vectors, текст chunks и metadata источников.

### Provider layer

Слой интеграций с embedding provider и answer provider.

Конкретные провайдеры выбираются через конфигурацию и остаются за application interfaces.

### External clients

Внешние интерфейсы для пользователя.

Они вызывают основной HTTP API, отображают ответ и источники, но не работают напрямую с Qdrant, Python helper или provider APIs.

## Критерии рабочего состояния

Проект находится в рабочем состоянии, если:

1. Локальное видео можно загрузить через `ingest`.
2. После ingest лекция доступна для поиска.
3. `POST /ask` принимает вопрос.
4. Система возвращает ответ по найденному контексту или fallback.
5. В ответе есть sources.
6. `rebuild` пересобирает индекс из сохранённых transcript-файлов.
7. Проект запускается по инструкции из `docs/quick-start.md`.
